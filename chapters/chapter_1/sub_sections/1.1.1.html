1.1.1 Problem Solving Approaches

Three Approaches

Top-Down

Breaking complex problems into smaller modules through decomposition

  • Stepwise refinement process
  • Tree structure representation
  • Used in procedural programming (C/Pascal)
  • Flow of control is downward

Bottom-Up

Combining small modules into complete solutions

  • Starts with fundamental components
  • Focus on integration testing
  • Used in OOP languages (Python/Java)
  • Eliminates redundancy

Modularisation

Creating independent system components

  • Modules work together
  • Main program calls subprograms
  • Common in OOP
  • Improves maintainability

Approach Comparison

Concept Top-Down Bottom-Up
Process Splitting (Decomposition) Merging (Integration)
Redundancy Contains redundant information Redundancy eliminated
Programming Languages Procedural (C) Object-Oriented (Python, Java)
Primary Use Documentation & Debugging Testing
